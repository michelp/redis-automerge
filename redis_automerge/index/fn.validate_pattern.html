pub fn validate_pattern(pattern: &str) -> Result<(), RedisError>
Expand description

Reject patterns that use Redis-glob metacharacters the in-tree matcher does not implement. The matcher in IndexConfig::matches_pattern only honors *; if a user configures user[12]:* expecting Redis-style glob semantics, the match would silently fail. Rather than vendor a full glob grammar, this gate makes the supported subset loud at configure-time. Audit #25.

Returns Err if pattern is empty or contains ?, [, ], or \.
